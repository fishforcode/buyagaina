# 订阅消息配置指南

## 📋 概述

智能提醒系统需要使用微信小程序的订阅消息功能来向用户发送临期提醒。

---

## 🔧 步骤一：申请订阅消息模板

### 1.1 登录微信公众平台
访问：https://mp.weixin.qq.com/

### 1.2 进入订阅消息管理
1. 进入「功能」→「订阅消息」
2. 点击「选用」

### 1.3 选择或创建模板

**方案A：使用公共模板**
搜索以下关键词：
- 「临期提醒」
- 「过期提醒」
- 「物品过期」

**方案B：自定义模板**
如果公共模板不满足需求，可以申请自定义模板。

### 1.4 推荐模板配置

#### 模板内容示例
```
物品：{{thing1.DATA}}
有效期至：{{time2.DATA}}
剩余天数：{{number3.DATA}}
位置：{{thing4.DATA}}
备注：{{thing5.DATA}}
```

#### 模板字段说明
| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| thing1 | 事物 | 物品名称 | 「牛奶」 |
| time2 | 时间 | 有效期至 | 「2026-02-11」 |
| number3 | 数字 | 剩余天数 | 「2」或「0」 |
| thing4 | 事物 | 位置 | 「冰箱」 |
| thing5 | 事物 | 备注 | 「常温保存」或「无」 |

---

## 🔑 步骤二：获取模板ID

申请通过后，在「我的模板」中可以看到模板ID，格式类似：
```
ABCDEF1234567890
```

**重要**：复制这个模板ID，用于配置代码。

---

## 💻 步骤三：配置代码中的模板ID

### 3.1 云函数配置
编辑文件：`cloudfunctions/checkExpiry/index.js`

找到第 117 行附近：
```javascript
// 订阅消息模板ID
const TEMPLATE_ID = 'YOUR_TEMPLATE_ID';
```

将 `YOUR_TEMPLATE_ID` 替换为你的实际模板ID，例如：
```javascript
const TEMPLATE_ID = 'kjNYfJVcrdGMfxi9HmS21LmpDfy0_syMUaewc2STCKs';
```

### 3.2 前端配置
编辑文件：`miniprogram/pages/index/index.js`

找到第 185 行附近和第 219 行附近：
```javascript
// TODO: 替换为实际的模板ID
var templateId = 'YOUR_TEMPLATE_ID';
```

同样替换为实际模板ID，例如：
```javascript
var templateId = 'kjNYfJVcrdGMfxi9HmS21LmpDfy0_syMUaewc2STCKs';
```

**注意**：这个位置有两处，都需要替换。

## 🚀 步骤四：上传云函数

### 4.1 安装依赖
在云开发控制台或本地运行：
```bash
cd cloudfunctions/checkExpiry
npm install
```

### 4.2 上传云函数
1. 打开微信开发者工具
2. 右键点击 `cloudfunctions/checkExpiry` 文件夹
3. 选择「上传并部署：云端安装依赖」

### 4.3 配置定时触发器
上传成功后，在云开发控制台：

1. 进入「云开发」→「云函数」
2. 找到 `checkExpiry` 函数
3. 点击「配置」→「触发器」
4. 添加两个触发器：
   - **触发器1**：午饭前提醒
     - 类型：定时触发器
     - Cron 表达式：`0 0 11 * * * *`
     - 说明：每天 11:00 触发

   - **触发器2**：晚饭前提醒
     - 类型：定时触发器
     - Cron 表达式：`0 0 17 * * * *`
     - 说明：每天 17:00 触发

## 🎯 Cron 表达式说明

定时触发器使用 Cron 表达式：

```
秒 分 时 日 月 星期 年
0  0  11 * * * *
```

- `0 0 11 * * * *` → 每天 11:00
- `0 0 17 * * * *` → 每天 17:00
- `0 30 9 * * * *` → 每天 9:30

## ❓ 常见问题

### Q1: 订阅消息发送失败？
**A**: 检查以下几点：
1. 模板ID是否正确
2. 用户是否已授权
3. 云函数是否配置了订阅消息权限
4. 查看云函数日志，查看具体错误信息

### Q2: 用户拒绝授权后如何重新授权？
**A**: 用户可以：
1. 在小程序设置中手动开启订阅消息
2. 或者在「我的」→「设置」中重新授权

### Q3: 可以修改提醒时间吗？
**A**: 可以。修改云函数触发器的 Cron 表达式即可。
- 修改提醒天数：在云函数中修改 `reminderDays` 默认值
- 修改触发时间：修改定时触发器的 Cron 表达式

### Q4: 订阅消息有次数限制吗？
**A**: 订阅消息没有严格的发送次数限制，但建议：
- 每个用户每天最多发送 2-3 条
- 避免重复提醒同一物品（代码已实现去重）

### Q5: 如何查看订阅消息发送记录？
**A**: 在微信公众平台 →「功能」→「订阅消息」→「消息记录」中查看。
