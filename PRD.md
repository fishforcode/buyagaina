# 物品管理小程序 PRD

## 1. 产品概述

### 1.1 产品定位
一款帮助家庭管理储物空间和物品的智能小程序，专注于解决物品过期、遗忘购买等问题，提供便捷的物品追踪和智能提醒功能。

### 1.2 核心价值
- **可视化存储管理**：将储物空间（冰箱、橱柜等）数字化，一目了然掌握物品分布
- **智能临期提醒**：自动检测临期物品，及时推送提醒，减少浪费
- **家庭协作**：支持家庭成员共享管理，实现家庭物品统一管理
- **数据洞察**：提供消耗统计和采购建议，优化家庭消费

### 1.3 目标用户
- 家庭主妇/主夫
- 合租室友
- 有储物管理需求的个人用户

**功能描述**：

#### 2.2 空间创建
**页面路径**：`create_location`

**功能描述**：
- 创建新的储物空间（如冰箱、橱柜、抽屉等）
- 支持设置空间名称、图标、备注

**关键点**：
- 数据写入 locations 集合

#### 2.3 空间详情与物品展示
**页面路径**：`location`

**功能描述**：
- 展示指定空间下的所有物品列表
- 支持下拉刷新
- 返回首页时自动刷新数据
- 支持添加物品入口

**关键点**：
- 关联查询 items 集合（过滤 locationId）
- 利用 onShow 生命周期自动刷新

- 按保质期排序，临期物品高亮

---

#### 2.4 物品添加
**页面路径**：`create_item`

**功能描述**：
- 为指定空间添加物品
- 支持输入物品名称、数量、保质期、备注

**关键点**：
- 数据写入 items 集合，关联 locationId
- 表单校验（必填项、数量合法性、日期合理性）
- 添加成功后返回空间详情页

---

#### 2.5 数据与后端
**技术实现**：
- 使用微信云开发环境
- 数据库集合：`locations`、`items`

**数据库设计**：
```
locations 集合
{
  _id: string,
  name: string,           // 空间名称
  icon: string,           // 空间图标
  remark: string,         // 备注
  createTime: timestamp,   // 创建时间
  creatorId: string       // 创建者ID
}

items 集合
{
  _id: string,
  name: string,           // 物品名称
  quantity: number,       // 数量
  expiryDate: timestamp,  // 保质期
  remark: string,         // 备注
  locationId: string,     // 所属空间ID（外键）
  createTime: timestamp,   // 创建时间
  creatorId: string       // 创建者ID
}
```

**权限设置**：
- 所有用户可读
- 仅创建者可写

### 阶段三：智能提醒系统（规划中 📋）

#### 2.8 智能临期提醒
**技术实现**：云函数 `checkExpiry`

**功能描述**：
- 自动扫描所有临期物品
- 发送订阅消息提醒用户
- 支持自定义提醒天数（默认 3 天）

**临期提醒规则**：

| 级别 | 状态 | 条件 | 消息样式 |
|------|------|------|----------|
| 🔴 红色 | 已过期 | 过期当天 | 【紧急提醒】 |
| 🟡 黄色 | 临期 | 保质期前 3 天内 | 【临期提醒】 |

**消息发送频率**：
- 每天 2 次
  - 午饭前 1 小时（如 11:00）
  - 晚饭前 1 小时（如 17:00）

**消息发送方式**：
- 方案A（采用）：每条临期物品发送一条消息
- 消息模板字段：提醒类型、物品名称、过期时间、所在空间

**关键点**：
- 配置云函数双定时触发器（11:00、17:00）
- 设计订阅消息模板
- 用户授权订阅消息
- 提醒天数可自定义（通过数据库配置，默认 3 天）
- 避免同一天重复提醒同一物品

**数据库扩展**：

`user_settings` 集合（新增）：
```
{
  _id: string,
  _openid: string,           // 用户openid
  reminderDays: number,      // 提前提醒天数（默认3）
  reminderEnabled: boolean,  // 是否开启提醒（默认true）
  updateTime: timestamp
}
```

`items` 集合（可选扩展）：
- 添加 `lastRemindedDate` 字段，记录最后提醒日期，避免重复提醒

**订阅消息模板设计**：
```
模板字段：
- thing1：提醒类型（紧急提醒 / 临期提醒）
- thing2：物品名称
- time3：过期时间
- thing4：所在空间

消息内容示例：
【临期提醒】
物品：牛奶
过期时间：2026-02-11
位置：冰箱
```

**实现步骤**：
1. 创建云函数 `checkExpiry`
2. 配置双定时触发器（每天 11:00、17:00）
3. 创建 `user_settings` 集合并初始化默认设置
4. 申请订阅消息模板
5. 实现扫描逻辑：查询 items 集合中 expiryDate 在提醒范围内的物品
6. 按用户分组，对每个临期物品发送订阅消息
7. 记录提醒状态（避免重复提醒）
8. 前端添加用户授权订阅消息引导（简单入口，无需完整设置页面）
